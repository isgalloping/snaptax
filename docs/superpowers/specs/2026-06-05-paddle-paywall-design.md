# Paddle 付费墙与换机提醒 — PRD 修订设计

**日期：** 2026-06-05  
**状态：** 已批准  
**影响文档：** `docs/prd/0.0.1.md` §2.3、§2.4、§3、§4  

---

## 背景

原 PRD §2.4.3 采用 Apple Pay / Google Pay 原生支付，且导出成功后按钮变为 `Exported ✓`，无法重复导出。修订目标：

- 支付渠道改为 **Paddle 一次性收费**（Overlay 浮层，不离开 App）
- 按报税季付费，本季内 **无限次重复导出**
- 在软引导、设置页、Paywall 三处强调 **换手机需 Google 登录，否则数据丢失**

## 已确认的产品决策

| 决策点 | 选择 |
|--------|------|
| 付费权益范围 | **A — 按报税季**：付一次 = 当前报税季内该 Google 账号下全部小票可无限次导出；新拍小票自动纳入 |
| 换机提醒位置 | **B**：§2.3 软引导横条 + 设置页 Google 区常驻小字 + Paywall 付款前警告 |
| Paddle 结账方式 | **A — Paddle Overlay**：App 内浮层完成支付，不跳转离开 Snap1099 |

## Paddle 付费墙（§2.4.3）

### 定价与权益

- **价格：** $49.00 / One-Time for 2026 Tax Season（一次性，非订阅）
- **权益：** 当前报税季（1–4 月）内，绑定 Google 账号下的 **全部小票** 可 **无限次重复导出**
- **新小票：** 本季内新拍摄的小票自动纳入已购权益，无需再付
- **跨季：** 下一报税季（如 2027 Tax Season）需 **重新购买** 新一季导出权

### 支付流程

1. Google 登录（硬拦截，若尚未绑定）
2. 底部滑出 Paywall 面板
3. 用户阅读换机警告 → 点击 **`Pay $49 with Paddle`**
4. **Paddle Overlay** 浮层唤起（支持信用卡 / Apple Pay 等 Paddle 渠道）
5. 支付成功 → 关闭 Overlay → 唤起系统分享（Email / WhatsApp）

### Paywall 面板结构

1. 价格 + 卖点（IRS 标准 Excel，发 CPA / 导入 TurboTax）
2. **换机警告行**（高对比黄字，不可折叠）：*「换手机前请用 Google 登录，否则本地数据会丢失」*
3. 主按钮：**`Pay $49 with Paddle`**
4. 无「Apple Pay / Google Pay 原生按钮」表述；统一经 Paddle 完成

### 已付费用户（本季）

- 导出按钮文案：**`Export Again`**（或 **`Re-export Tax Pack`**），**非** `Exported ✓` 终态
- 点击 **不再弹出 Paywall**，直接生成 Excel 并分享
- 设置页显示：`2026 Tax Season · Paid ✓`

## 换机 / 数据丢失提醒（§2.3 + §2.4）

| 位置 | 未登录 | 已登录 |
|------|--------|--------|
| **§2.3 软引导横条** | *「怕数据丢失？用 Google 账号一键保存（换手机必登录）」* | 横条已消失（已绑定） |
| **设置页 Google 区** | *「未登录 · 换手机数据会丢失」* 常驻小字 | *「已登录 · xxx@gmail.com · 数据已云端保存」* |
| **Paywall 面板** | 付款前必看警告行（见上） | 已付费用户不进入 Paywall |

---

## 异常场景（§3 增补）

| 场景 | 解法 |
|------|------|
| Paddle 支付失败或取消 | 底部红字提示，停留 Paywall，可重试 |
| 已付费用户换设备 | 新设备 Google 登录 → 识别本季权益 → 直接 Export Again |
| 未登录换机 | Ghost 本地数据无法恢复；产品已通过三处提醒尽告知义务 |
